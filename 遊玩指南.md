## 1. 這是什麼遊戲？

- **類型**：文字辯論 RPG，風格接近《極樂迪斯科》（Disco Elysium）——你有內心聲音（思維卡）、技能檢定、以及由 **AI 扮演** 的查拉圖斯特拉。
- **主線目標**：在哲學辯論中**勝過查拉圖斯特拉**（論點說服他，或經 AI 裁判正式判定你贏）。
- **副線目標**：每局隨機抽 2 個「肉鴿挑戰」（例如連續檢定成功 3 次、6 輪內獲勝等）。
- **重要**：幾乎所有查拉圖斯特拉的台詞、四條「回應路線」檢定成功後的玩家台詞、以及**結算裁判報告**，都依賴 **OpenRouter API**。沒有 API Key 無法正常進行遊戲。

## 2. 開始前準備

### 2.1 如何開啟

1. 用 Chrome / Edge / Firefox 開啟 `index.html`（雙擊或拖進瀏覽器）。
2. 建議全螢幕；視窗太窄時左右側欄會隱藏，但核心對話區仍可用。

### 2.2 OpenRouter API Key（必備）

1. 到 [OpenRouter](https://openrouter.ai/) 註冊並建立 API Key（格式通常為 `sk-or-v1-...`）。
2. 在**角色建立畫面**的「API Key」欄位貼上。
3. 按 **「🔌 測試連接」** 確認可用。
4. Key **只存在你的瀏覽器**（`localStorage`），不會上傳到別人的伺服器（除非你自行把存檔匯出給別人）。

### 2.3 模型

- 預設模型：`openrouter/owl-alpha`（可在角色建立時改）。
- 結算辯論會送較長的完整紀錄給 AI，若失敗可換上下文更長、或更穩定的模型。

### 2.4 可選：程式碼內建 Key

`index.html` 頂部有 `const API_KEY = '...'`。若你在開發，可填入預設 Key；**一般玩家請用創角畫面輸入**，不要把真 Key 提交到 Git。

## 3. 從開機到第一局

畫面依序為：

```
開場標題 → 存檔選擇 → 角色建立 → 辯論主題 → 正式遊戲
```

### 3.1 開場

- 按 **「✦ 進入洞穴 ✦」** → 存檔畫面。

### 3.2 存檔

- **新遊戲**：進角色建立。
- **讀檔**：點已有存檔格，會還原技能、思維、對話紀錄、辯論進度等。
- **匯入/匯出**：JSON 檔，可備份或換電腦。

### 3.3 角色建立

| 項目 | 說明 |
|------|------|
| **名字** | 顯示在對話中，最多 20 字。 |
| **技能** | 邏輯、意志、共情、修辭；各項預設 3，共有 **12 點** 可分配，**單項最高 15**。 |
| **原型 Archetype** | 選預設（哲學家/叛逆者/神秘主義者/詩人）或輸入描述讓 AI 生成；原型會讓某一技能 **+2**（疊在基礎值上）。 |
| **思維草稿** | 從隨機 **10 張** 思維卡中選 **3 張** 作為開局內閣；可 **「🔄 重抽 10 個」**。 |
| **開始** | 填好 API Key、選滿 3 張思維後，按 **「✦ 開始辯論 ✦」**。 |

### 3.4 辯論主題

- **主目標**固定：勝過查拉圖斯特拉。
- **副挑戰**：本局隨機 2 個（顯示在畫面兩側小卡）。
- 可 **自訂主題**（輸入框），或 **「🎲 隨機主題」**（從內建尼采相關題目池抽）。
- 按 **「✦ 帶著這個主題進入洞穴 ✦」** 進入遊戲；查拉圖斯特拉會依主題開場。

### 3.5 開局懲罰（陣營狂熱）

若在創角時，某一派系思維已 **≥4 張**（且沒有並列最高），視為「一開始就過度信奉某主義」：

- 該派 **招牌技能 +2**（見下文）。
- **初始好感 −10**（查拉圖斯特拉更鄙視「羊群式信徒」）。

## 4. 一局辯論怎麼玩（核心迴圈）

每一「輪」大致如下：

```
查拉圖斯特拉發言（AI）
    ↓
出現「四條回應路線」（邏輯／意志／共情／修辭）
    ↓
你選一條 → 擲骰檢定（難度一開始隱藏）
    ↓
├─ 失敗 → 自己在下方輸入框親手打字反駁
└─ 成功 → AI 替你寫一句話 → 再按「辯論」送出
    ↓
查拉圖斯特拉再次回應（AI）……循環
```

### 4.1 兩種輸入方式

1. **回應路線（推薦）**  
   - 查拉圖斯特拉每段話結束後，中間會出現 **「◈ 回應路線」** 四格按鈕。  
   - **先點選** 一條路線 → 系統擲 `1d20 + 該技能值`，與隱藏目標值比較。  
   - **檢定通過後**，AI 才生成你的台詞；生成完再點該按鈕或 **「辯論」** 送出。  
   - **檢定失敗**：該路線鎖死，請在**最下方輸入框**自己寫反駁（其他路線仍可試）。

2. **自由輸入**  
   - 也可不走路線，直接在輸入框寫內容按 **「辯論」**。  
   - 系統仍會分析你的文字（敵意、哲學深度等），影響好感、辯論優勢、怒氣，並觸發 AI 回應。

### 4.2 一輪一張思維卡

- 透過檢定成功等方式 **領悟新思維** 時，**同一回合只能獲得 1 張**。
- 想再抽卡：先 **發言**（按辯論推進回合），下一輪才能再領悟或使用 **「🎲 領悟新思維」**（若該回合尚未獲卡）。

### 4.3 等待 AI 時

- 按鈕會暫時禁用（`isProcessing`），請勿連點。
- 右下角 **蘇式 AI 面板**（可拖曳、滾輪縮放）：  
  - **THINK**：正在呼叫 OpenRouter  
  - **DISPLAY**：本地打字動畫（非 AI）  
  - **ALARM**：錯誤或 Key 問題

## 5. 四項技能與檢定

| 技能 | 英文 | 擅長 |
|------|------|------|
| **邏輯** | Logic | 拆漏洞、概念、矛盾 |
| **意志** | Volition | 站穩立場、承擔代價、對抗壓迫 |
| **共情** | Empathy | 觸及他對人類的複雜情感 |
| **修辭** | Rhetoric | 詩意、比喻、語言魅力 |

### 5.1 回應路線檢定

- 公式：`1d20 + 當前技能值` ≥ **隱藏目標值** 則通過。  
- **大成功**：擲出 20 通常可過（除非規則另有 fumble 處理）。  
- **大失敗**：擲出 1 通常直接失敗。  
- 目標值 **不會事先顯示**；擲完後系統訊息會揭露「目標 X」。  
- 難度會隨 **你的技能、查拉圖怒氣、已進行回合數** 升高（高技能也不會一路輕鬆碾壓）。

### 5.2 隱藏難度大致範圍

- 目標值約在 **16～27** 之間（由程式 `hiddenRouteDC` 計算）。  
- 怒氣越高、回合越多，越難。

### 5.3 檢定成功／失敗的常見後果

| 結果 | 常見效果 |
|------|----------|
| 路線成功 | 辯論優勢 +5、怒氣 +2、可能領悟新思維 |
| 路線失敗 | 辯論優勢 −2、怒氣 +4、需手動輸入 |

另有一種 **主題關鍵字檢定**（`doAISkillCheck`）：當你的發言被判定觸及特定哲學主題時，會在 AI 回應後追加一次技能檢定，影響好感與優勢，但**不會再打斷對話插入第二段罐頭文**。

## 6. 思維內閣（肉鴿卡牌）

### 6.1 基本概念

- 左側 **「思維內閣」** 最多 **10 張** 思維卡。
- 每張卡屬於四大陣營之一，並帶 **屬性修正**（例如邏輯 +1、修辭 −1）與一段 **能力描述**（Flavor，影響玩法氛圍與 AI 對你的印象）。
- 四陣營符號：**康米 ★ · 社民 ✿ · 法西斯 ▲ · 自由 ◇**

### 6.2 如何獲得新卡

1. **技能檢定成功**（辯論中）→ 有機會自動領悟一張（每回合最多一張）。  
2. **「🎲 領悟新思維」** 按鈕 → 從尚未擁有的卡池隨機抽一張（仍受每回合一次、內閣已滿、腦霧冷卻限制）。

### 6.3 內化（新卡代價）

- 新卡進入內閣後會 **內化 2 回合**：  
  - 在內化完成前，**主要相關技能 −1**（固著感）。  
  - 2 回合後才套用卡面上的正式加成。

### 6.4 遺忘（✕）

- 點卡片右上角 **✕** 可遺忘該思維。  
- **代價**：好感 **−4**；**3 回合內無法**再領悟新卡（腦霧冷卻）。  
- 內閣滿了時必須先遺忘才能裝新卡。

## 7. 陣營、狂熱與超技

### 7.1 陣營計數

- 左側內閣上方會顯示各派持有張數。

### 7.2 狂熱（同派 ≥4 張，且無並列最高）

- 該派 **招牌技能 +2**：  
  - 康米 → 邏輯  
  - 社民 → 共情  
  - 法西斯 → 意志  
  - 自由 → 修辭  
- 代表你的思想過度單一；查拉圖斯特拉會更警惕（開局若已狂熱還會 **好感 −10**）。

### 7.3 陣營超技（同派 ≥5 張，且無並列）

- 工具列 **「✦ 陣營超技」** 亮起後可使用。  
- **每局只能用一次**。  
- 效果因派而異（改技能、好感、戰場視覺、遺忘冷卻等），並切換洞穴 **戰場模式**（赤色/白晝/鐵色/霓虹等）。

| 陣營 | 超技名稱（摘要） |
|------|------------------|
| 康米 | 總罷工：邏輯+2、共情+2、好感+12 |
| 社民 | 白晝福利國：共情+3、好感+15、清除遺忘冷卻 |
| 法西斯 | 鋼鐵閱兵：意志+4、修辭+1、好感−8 |
| 自由 | 市場黎明：邏輯+2、修辭+3、共情−1 |

### 7.4 日光介面

- **「☀ 日光介面」**：純 UI 切換（亮色系），隨時可開關，與超技無關。

## 8. 怒氣、好感、辯論優勢

### 8.1 查拉圖斯特拉的怒氣（0～100）

- 右側顯示；越高他越尖銳、越難討好，檢定也可能更難。  
- 挑釁、檢定失敗等會 **升高**；論點刺中他會 **升高一點**（被激怒）或依情境變化。  
- 怒氣 **≥60** 時系統提示：可能露出 **破綻**（內建弱點提示，供你攻擊）。  
- 怒氣 **≥70** 時頭像會有威壓動畫。

### 8.2 好感 disposition（0～100）

- 影響他對你的態度與部分結局敘事。  
- 長文、哲學性發言、檢定成功通常 **+**；挑釁、超技（法西斯）、遺忘思維等會 **−**。  
- 若曾跌破 35，會影響副挑戰 **「鐵壁意志」**（需正式獲勝且從未跌破 35）。

### 8.3 辯論優勢 debateScore（HUD 進度條）

- 右側 **「辯論優勢 %」** 由 `debateScore` 換算（0～100%）。  
- 好發言、路線成功、主題檢定成功等會 **增加**；失敗會 **減少**。  
- 當優勢 **≥85%** 或好感 **≥88** 時，系統會提示 **「已達臨界」**——這**不是**正式勝利，只是「可以結算了」。  
- **正式獲勝**請看下一節。

## 9. 怎樣算「贏」？結算與裁判報告

### 9.1 兩種「贏」的層次

| 類型 | 條件 | 意義 |
|------|------|------|
| **臨界優勢** | 辯論優勢 ≥85% 或好感 ≥88 | 提示你可以結算；**可繼續辯論** |
| **正式勝利** | 按 **「⚖ 結算辯論」**，AI 裁判判定「勝者：玩家」 | 解鎖勝利結局、副挑戰判定、裁判報告 |

### 9.2 結算辯論（必做才有報告）

1. 工具列按 **「⚖ 結算辯論」**（至少需要 **2 句以上** 對話紀錄）。  
2. AI 會讀取整場紀錄，輸出 **裁判報告**（勝者、總評、四段分析）。  
3. 若判定你贏 → 彈出 **「辯論勝利」** 全螢幕，**報告自動展開**。  
4. 若判定查拉圖斯特拉贏 → **「辯論落敗」**，仍可看報告學哪裡輸了。

### 9.3 結局畫面按鈕

| 按鈕 | 作用 |
|------|------|
| **↩ 繼續辯論** | 關閉結局層，**回到對話**（已正式獲勝也可繼續聊） |
| **📜 展開／收合報告** | 切換報告區顯示 |
| **📜 裁判報告**（工具列） | 隨時再打開結局畫面看上次報告 |
| **⟳ 重開這局** | 重新整理頁面 |
| **← 返回選單** | 回存檔畫面 |

> **注意**：若只打到 85% 優勢卻**從未按結算**，不會有裁判報告，也不會算 `debateWon` 正式勝利。

### 9.4 副挑戰（本局側邊目標）

每局隨機 2 個，例如：

- 連續 3 次技能檢定成功  
- **6 輪內**正式獲勝（`debateWon`）  
- 觸及 5 個以上哲學主題關鍵字  
- 正式獲勝且好感從未低於 35  
- 從極低好感大幅回升  
- 僅用邏輯/共情通過檢定 3 次以上  

完成與否會顯示在結局畫面的小標籤上。

## 11. 存檔與 API

### 11.1 存檔內容

- 角色名、技能、思維內閣、對話歷史、好感、怒氣、辯論主題、是否已勝、**上次裁判報告** 等。  
- 遊戲中可用右側 **儲存**（若有按鈕）或選單流程存至本地 slot。

### 11.2 API 費用

- 每次查拉圖回應、路線台詞生成、結算，都會消耗 OpenRouter 額度。  
- 結算一次送 **最多約 12000 字** 的辯論紀錄，費用較高，請在優勢夠高或想結束時再按。

### 11.3 篇幅滑桿（AI 面板）

- **查拉圖篇幅 / 玩家篇幅**（1～10）：影響 AI 生成句數與 token 上限；過高可能更慢、更易觸發截斷（程式有自動續寫機制，但仍建議適中）。

## 12. 給 AI Agent 的操作備忘

若你是自動化 Agent（例如透過瀏覽器操作、或協助玩家代打），請遵守以下機制，避免「以為贏了卻卡住」或「沒有報告」。

### 12.1 檔案與執行

- **唯一遊戲檔**：`index.html`  
- **無後端**：所有狀態在瀏覽器 `state` 物件與 `localStorage`  
- **必須有 API Key** 才能：`processInput`、路線 `generatePlayerLine`、`settleDebate`

### 12.2 建議操作順序（一局）

1. 開啟 `index.html` → `showSaveSlots` → `newGame(slot)` → 填創角 → `startGame()` → `beginRun()`  
2. 等待開場白與 `offerDialogueRoutes` 出現  
3. 每回合優先：點 **一個未失敗的 choice-btn** → 等檢定與 AI 台詞 → 再送出  
4. 監聽 `state.isProcessing`；為 `true` 時不要點 `send-btn`  
5. 當 `debateScore >= 85` 或系統出現臨界提示 → 可選繼續辯論或呼叫 `settleDebate()`  
6. 結算後：讀 `#re-report` 內 HTML；若要繼續玩 → `dismissRunEnd(true)` 或點「↩ 繼續辯論」

### 12.3 關鍵 `state` 欄位

| 欄位 | 意義 |
|------|------|
| `isProcessing` | AI 忙碌中，勿送出新訊息 |
| `debateScore` | 辯論優勢（0～100+，HUD 用 min(100, score)） |
| `debateWon` | 僅在 **結算判定玩家贏** 後為 true |
| `debateClinch` / `_clinchNotified` | 達臨界優勢的提示用 |
| `lastSettlementReport` | 裁判報告 HTML |
| `_pendingChoices` | 四條路線物件陣列 |
| `thoughts` | 內閣 id 列表 |
| `forgetCooldown` | 遺忘後剩餘回合 |
| `ultraUsed` | 超技是否已用 |
| `zaraAnger` | 怒氣 0–100 |

### 12.4 關鍵函式（除錯用）

| 函式 | 用途 |
|------|------|
| `sendMessage()` | 自由輸入送出 |
| `tryDialogueChoice(i)` | 第 i 條路線（0–3） |
| `settleDebate()` | AI 裁判結算 |
| `showRunEnd(win)` | 顯示結局層 |
| `dismissRunEnd(true)` | 關閉結局並恢復輸入 |
| `openSettlementReport()` | 僅查看報告 |
| `checkObjectives()` | 臨界提示（不應再自動鎖死遊戲） |

### 12.5 Agent 常犯錯誤

1. **把 85% 優勢當正式勝利** → 仍需 `settleDebate()` 才有 `debateWon` 與報告。  
2. **結局層擋住畫面以為壞掉** → 點「繼續辯論」或工具列「裁判報告」。  
3. **沒 API Key 仍點路線** → 檢定成功後台詞生成失敗，需改用手動輸入。  
4. **一回合連按兩次領悟** → 第二張會被 `_gainedThisRound` 擋下。  
5. **內閣滿了仍想抽卡** → 先 `forgetThought(id)`。

## 13. 常見問題

**Q：為什麼查拉圖斯特拉不說話 / 一直轉圈？**  
A：檢查 API Key、模型名稱、OpenRouter 餘額；看右下角 ALARM 燈與系統紅字。

**Q：我優勢 100% 了為什麼沒贏？**  
A：臨界 ≠ 裁判勝利。請按 **「⚖ 結算辯論」**。

**Q：贏了但看不到報告？**  
A：必須走結算流程；結局裡按 **「📜 裁判報告」** 或工具列同名按鈕。若曾舊版自動彈窗無報告，請 **Ctrl+F5** 強制重新整理後用新版再結算一次。

**Q：贏了還能不能繼續聊？**  
A：可以。結局畫面選 **「↩ 繼續辯論」**。

**Q：可以離線玩嗎？**  
A：不行。核心對話與路線台詞皆需 OpenRouter。

**Q：思維卡會不會讓 AI 一直念卡名？**  
A：Prompt 要求 AI 只感知傾向，不要逐張念內閣清單；但仍可能偶爾提到你的思想陣營。
